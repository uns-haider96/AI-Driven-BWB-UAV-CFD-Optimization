# GitHub Upload Guide

Use this repository folder as the final public version.

## Repository Name

Use this exact GitHub repository name:

```text
AI-Driven-BWB-UAV-CFD-Optimization
```

Final public URL:

```text
https://github.com/uns-haider96/AI-Driven-BWB-UAV-CFD-Optimization
```

## Create the Repository

On GitHub:

1. Click `New repository`.
2. Repository name: `AI-Driven-BWB-UAV-CFD-Optimization`.
3. Set visibility to `Public`.
4. Keep these OFF:
   - Add README
   - Add .gitignore
   - Add license
5. Click `Create repository`.

This package already contains `README.md`, `.gitignore`, and `LICENSE`.

## What to Upload

Upload the contents of this folder, not the ZIP file itself.

Upload:

- `README.md`
- `START_HERE_FOR_REVIEWERS.md`
- `PUBLICATION.md`
- `CITATION.cff`
- `LICENSE`
- `.gitignore`
- `ENGINEERING_SCOPE_AND_INTERPRETATION.md`
- `assets/`
- `docs/`
- `methodology/`
- `results/`
- `workflow/`
- `code/`

Do not upload:

- Conference paper PDF
- Publisher-formatted journal PDF
- Journal manuscript DOCX
- Raw ANSYS/Fluent solver files
- Private drafts
- `_review_before_public_upload/`

## If GitHub Website Upload Has Folder Problems

Upload in this order:

1. Root files first: README, PUBLICATION, LICENSE, etc.
2. `assets/` folder
3. `docs/` folder
4. `methodology/` folder
5. `results/` folder
6. `workflow/` folder
7. `code/` folder

If a folder opens instead of uploading, open that folder locally, select the files inside it, and upload them manually.
